# Contributing to Aneirin

Thanks for your interest in Aneirin. Feedback from real users is the most
valuable thing for this project, and there are good ways to help even though
the core is closed source.

## Please read this first

Aneirin is a **closed source product**. The optimizer engine ships only as a
compiled binary, and we do **not** accept pull requests that change the engine
code. This keeps the product focused and protects the work behind it.

That said, there is a lot you can do that really helps.

## Ways to contribute

### 1. Report a bug (very welcome)

If something does not work, or Aneirin ever changes an answer you get, please
open a **Bug report** issue. Good bug reports include:

* Your operating system (for example Windows 11).
* Which tool you were using (Claude Code or GitHub Copilot, CLI or VS Code).
* What you expected, and what happened instead.
* Any message shown in the terminal or dashboard.

Please do **not** include private prompts, code, or secrets in a bug report.

### 2. Request a feature or a higher free limit

Have an idea, or hit the free daily limit? Open a **Feature request** issue and
tell us what you need. We especially want to hear from small users about the
free tier limit. If enough people ask, we are happy to raise it.

### 3. Ask a question

Not sure how something works? Open an issue with your question, or start a
discussion. No question is too small.

### 4. Improve the docs

Spotted a typo or something unclear in the README or the guides? A small pull
request to the documentation is welcome. (Docs only. The engine stays closed
source.)

## Reporting a security problem

Please do **not** open a public issue for security problems. See
[SECURITY.md](SECURITY.md) for how to report privately.

## A note on tone

Please be kind and clear. See our [Code of Conduct](CODE_OF_CONDUCT.md). English
does not have to be perfect, we are happy to help you get your point across.

Thank you for helping make Aneirin better.
